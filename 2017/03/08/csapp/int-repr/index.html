Representation And Manipulation Of Interger Types in C(Draft)

2017-03-08
#CSAPP
Note

Overview

Integral types in C language has two major characteristics:

  1. Width - determines how many continuous bits should form a number
  2. Signedness - decides how those bits should be intepretted

Representation

Mathematical Model

For ease of discussion, we may write any typed integer with width of w as a w-dimensional bit vector $\vec{x} = (x{w-1}, …, x{1}, x{0})$, where $x{i}$ denotes the value of the corresponding bit, being either 0 or 1.

Unsigned Integer

Unsigned integers in C are denoted by exactly its binary written form.
For formal definition, we have evaluation function $B2U{w}(\vec{x})$ which maps a bit vector to the number it represents:
$$
B2U{w}(\vec{x}) = \sum{i=0}^{w-1} x{i} \times 2^{i}
$$

Signed Integer

Signed integers in C are more tricky. It’s encoded with a technique called Two’s Complement.
Evaluation function $B2T{w}(\vec{x})$ describes how it works:
$$
B2T{w}(\vec{x}) = {-x{w-1} \times 2^{w-1}} + \sum{i=0}^{w-2} x_{i} \times 2^{i}
$$

Notably, there’re absolutely other ways to denote a signed integer such as One’s Complement and Sign-Magnitute. However, Two’s Complement has several advantages:

  • Ease Of Implementation
    Almost all arithmetic operations over Two’s Complement has identical behavior to that for unsigned integer, which gigantically simplifies the implentation for a computation device. Such representation even allows addition of a signed integer and an unsigned integer, which should yield the correct result as long as you assume both integers have the same signedness. Last but not least, it simplifies casting between different width, which should be explained later.
  • Uniqueness
    Unlike One’s Complement or Sign-Magnitute, in which the number of zero may have different representations(more specificly, $+0$ and $-0$), Two’s Complement encode zero as one unique zero vector. Though such encoding also makes range of representable range unsymmetric, which is not that intuitive.
  • Continuity
    This is a brilliant part of the encoding. Two’s Complement smartly ensures continuity when overflow occurs(i.e. the smallest negative value equals one plus the largest positive value).

Manipulation

Casting in Width

  • Truncating
    Truncating a number is easy. Simply discard the higher-order bits and the job is done. There should be no loss as long as the old value fit the new width. This is obvious for an unsigned integer, but for signed number, it becomes a little tricky but Two’s Complement is still smart enough to handle this.
    (explaination why Two’s Complement works)
  • Expansion

  1. zero expansion for unsigned

  2. sign expansion for signed

Casting in signedness

Actually, C does nothing but simply change the intepretation over the bit vector. Therefore, the transformation function is really simple.

  • When cast a unsigned integer signed:
    $$ U2T{w}(x) = B2T{w}(U2B_{w}(x)) $$
  • When cast a signed integer unsigned:
    $$ T2U{w}(x) = B2U{w}(T2B_{w}(x)) $$

Bitwise Operations

  • And/Or/Xor
  • Reverse
  • Shift

Arithmetics Operations

  • Addition/Subtraction same as binary number
    overflow and underflow
  • Multiplication/Division
  • Modulus
